#<a name="managing-media-services-entities-with-rest-api"></a>Správa médií služby entity pomocí rozhraní REST API

> [AZURE.SELECTOR]
- [ZBÝVAJÍCÍ](media-services-rest-manage-entities.md)
- [.NET](media-services-dotnet-manage-entities.md)

Microsoft Azure Media Services je služeb na základě ZBÝVAJÍCÍ založená na OData v3. Vzhledem k tomu můžete přidat, dotazu, aktualizovat a odstraňovat entity velmi stejným způsobem můžete na jinou službu OData. Výjimkami bude vyznačenými-li k dispozici. Další informace o OData najdete v článku [si přečtěte následující dokumentaci Open Data Protocol](http://www.odata.org/documentation/).

- Přidání osob 
- Dotazování entity 
- Výčet prostřednictvím velkých sad entity
- Aktualizace entity 
- Odstranění entity 

>[AZURE.NOTE] Když pracujete s Media Services REST API, platí následující omezení:
>
>Při přístupu k entit v Media Services, musíte nastavit konkrétní záhlaví pole a hodnoty v žádostech HTTP. Další informace najdete v tématu [nastavení pro médií služby REST API vývoj](media-services-rest-how-to-use.md).

>Po úspěšném připojení k https://media.windows.net, zobrazí se 301 přesměrování zadání jiného identifikátor URI médií služeb. Musíte provést následující volání nových URI způsobem popsaným v části [připojování ke službě médií pomocí rozhraní REST API](media-services-rest-connect-programmatically.md). 


##<a name="adding-entities"></a>Přidání osob

Každá entita aplikace Media Services přibude sadu entity, například prostředky, až žádost HTTP příspěvek.

Následující příklad ukazuje, jak vytvořit AccessPolicy.

    POST https://media.windows.net/API/AccessPolicies HTTP/1.1
    Content-Type: application/json;odata=verbose
    Accept: application/json;odata=verbose
    DataServiceVersion: 3.0
    MaxDataServiceVersion: 3.0
    x-ms-version: 2.11
    Authorization: Bearer http%3a%2f%2fschemas.xmlsoap.org%2fws%2f2005%2f05%2fidentity%2fclaims%2fnameidentifier=youraccountname&urn%3aSubscriptionId=2f84471d-b1ae-4e75-aa09-010f0fc0cf5b&http%3a%2f%2fschemas.microsoft.com%2faccesscontrolservice%2f2010%2f07%2fclaims%2fidentityprovider=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&Audience=urn%3aWindowsAzureMediaServices&ExpiresOn=1337067658&Issuer=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&HMACSHA256=dithjGvlXR9HlyAf5DE99N5OCYkPAxsHIcsTSjm9%2fVE%3d
    Host: media.windows.net
    Content-Length: 74
    Expect: 100-continue
    
    {"Name": "DownloadPolicy", "DurationInMinutes" : "300", "Permissions" : 1}

##<a name="enumerating-through-large-collections-of-entities"></a>Výčet prostřednictvím velkých sad entity

Při dotazování entity, je omezení na 1000 entity vrátit najednou, protože veřejné v2 ZBÝVAJÍCÍ omezení výsledků dotazu 1000 výsledky. Umožňuje **Přeskočit** a **horní** umožňuje zobrazit výčet prostřednictvím rozsáhlé kolekce entity. 

Následující příklad ukazuje, jak pomocí **přeskočte** a **Nahoře** přeskočte nejdřív 2000 úlohy a získejte další 1000 úlohy.  

    GET https://media.windows.net/api/Jobs()?$skip=2000&$top=1000 HTTP/1.1
    Content-Type: application/json;odata=verbose
    Accept: application/json;odata=verbose
    DataServiceVersion: 3.0
    MaxDataServiceVersion: 3.0
    x-ms-version: 2.11
    Authorization: Bearer http%3a%2f%2fschemas.xmlsoap.org%2fws%2f2005%2f05%2fidentity%2fclaims%2fnameidentifier=youraccountname&urn%3aSubscriptionId=2f84471d-b1ae-4e75-aa09-010f0fc0cf5b&http%3a%2f%2fschemas.microsoft.com%2faccesscontrolservice%2f2010%2f07%2fclaims%2fidentityprovider=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&Audience=urn%3aWindowsAzureMediaServices&ExpiresOn=1337078831&Issuer=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&HMACSHA256=suFkxhvPWxQVMjOYelOJfYEWkyTWJCBc02pF0N7NghI%3d

##<a name="deleting-entities"></a>Odstranění entity

Entity můžete odstranit v Media Services pomocí žádost HTTP odstranit. V závislosti na entitu může být pořadí, ve kterém můžete odstranit entity důležité. Například entity například prostředky požadovat, aby odvolání (nebo odstranění) všechny Locator, které odkazují na daný majetek před odstraněním majetku.

Následující příklad ukazuje, jak odstranit URL, která byla použita k nahrání souboru do úložiště objektů blob.

    DELETE https://media.windows.net/API/Locators('nb:lid:UUID:76dcc8e8-4230-463d-97b0-ce25c41b5c8d') HTTP/1.1
    Content-Type: application/json;odata=verbose
    Accept: application/json;odata=verbose
    DataServiceVersion: 3.0
    MaxDataServiceVersion: 3.0
    x-ms-version: 2.11
    Authorization: Bearer http%3a%2f%2fschemas.xmlsoap.org%2fws%2f2005%2f05%2fidentity%2fclaims%2fnameidentifier=youraccountname&urn%3aSubscriptionId=2f84471d-b1ae-4e75-aa09-010f0fc0cf5b&http%3a%2f%2fschemas.microsoft.com%2faccesscontrolservice%2f2010%2f07%2fclaims%2fidentityprovider=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&Audience=urn%3aWindowsAzureMediaServices&ExpiresOn=1337067658&Issuer=https%3a%2f%2fwamsprodglobal001acs.accesscontrol.windows.net%2f&HMACSHA256=dithjGvlXR9HlyAf5DE99N5OCYkPAxsHIcsTSjm9%2fVE%3d
